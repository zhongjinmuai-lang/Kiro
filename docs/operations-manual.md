# MU 框架 · 三级管控操作手册

> 面向**开发商**、**服务商**、**终端客户**三种运营角色的快速上手指南

---

## 一、开发商总后台（顶层集权）

### 1.1 登录
- 入口：`https://admin.mu-framework.com`
- 初始账号：`admin` / `mu_admin_2026`（请首次登录后立即修改）

### 1.2 核心操作

#### 服务商管理
- 入驻审核：`/admin/developer/providers` → 新建 → 审核通过 → 生效
- 分润规则：对每个服务商配置平台抽佣比例
- 启用/禁用：一键冻结服务商（级联影响其下所有客户）

#### 支付中台配置
1. 进入"支付中台 → 渠道管理"
2. 点击"准入新渠道"（选择微信/支付宝/聚合支付）
3. 填写接口密钥、回调地址、商户资质
4. 选择哪些服务商可使用该渠道（渠道授权）
5. 关闭某渠道 → 所有服务商/终端客户即时失效

#### 存储中台配置
1. "存储中台 → 厂商准入"
2. 勾选启用的供应商（阿里OSS / 腾讯COS / 七牛 / 华为OBS）
3. 设置全局默认配额、单文件大小上限
4. 是否允许服务商绑定自有存储

#### 通知中台配置
1. "通知中台 → 渠道准入"（短信/邮件/站内信等）
2. 配置短信/邮件接口密钥与签名
3. 定义全局消息模板（续费提醒、到期告警、系统公告等）
4. 是否允许服务商自定义品牌模板

#### 系统管理
- 框架热更新：`/admin/developer/system/hot-update` → 上传插件 → 在线灰度
- 灰度发布：指定某服务商先启用新功能，验证后全量
- 域名/SSL：自动申请 Let's Encrypt，或绑定自有证书

## 二、服务商SaaS管理后台（二级管控）

### 2.1 客户管理
- 新增终端客户：`/admin/provider/customers/new`
- 套餐授权：为客户分配可用版本、插件清单
- 配额分配：为客户分配 AI 调用次数、存储空间
- 启用/禁用：关闭客户访问（不影响数据）

### 2.2 支付管理
- **可用渠道**：仅列出开发商已准入且已授权给本服务商的渠道
- **绑定自有商户号**：平台允许的情况下可使用自有微信商户号
- **订单查看**：下属客户的所有订单
- **分润明细**：本服务商的佣金收入
- **提现申请**：提交财务审核

### 2.3 通知管理
- **模板微调**：在平台基础模板上调整文案（落款、签名）
- **客户通知规则**：为下属客户配置到期提醒、用量告警触发条件
- **渠道开关**：关闭某客户的某类通知

### 2.4 品牌定制
- 登录页 Logo / 主题色
- 公告栏内容
- 自定义域名（`provider1.mu-framework.com`）

## 三、终端客户业务后台（三级使用）

### 3.1 族谱管理
- **可视化展示**：世系图、关系树、分支图谱
- **AI 建档**：拍照上传老族谱 → AI 自动识别 → 人工校对 → 入库
- **导入导出**：Excel 批量导入 / PDF 打印导出
- **权限分级**：族长、分支管理员、普通族人
- **备份恢复**：一键备份到云端 / 指定时间点恢复

### 3.2 文件存储（受限使用）
- 在服务商分配的**配额内**上传照片、扫描件
- 支持在线预览、下载、归档
- 查看自身用量与剩余配额
- **不可**选择存储厂商、**不可**配置密钥

### 3.3 支付订阅（受限使用）
- 在服务商提供的**套餐内**续费、升级、购买插件
- 查看订单、发票、支付记录
- **不可**选择支付渠道、**不可**修改费率

### 3.4 通知接收（受限使用）
- 接收套餐到期、续费、权限变更通知
- 管理站内信已读/未读
- 订阅开关（只能控制是否接收，不能改模板）

### 3.5 移动端
- UniApp X 多端：微信/抖音/支付宝/小红书/鸿蒙 小程序 + Android/iOS App + H5
- 扫码上传老族谱、一键寻根溯源
- 家族动态、家族公告

## 四、三级联动示例

### 示例 1：开发商关闭微信支付
```
[Developer] 关闭 wechat 渠道
    ├─ [Provider A] 后台微信支付选项立即消失
    │      └─ [Customer A1] 订单页不再展示微信支付
    │      └─ [Customer A2] 订单页不再展示微信支付
    └─ [Provider B] 后台微信支付选项立即消失
           └─ [Customer B1] 订单页不再展示微信支付
```

### 示例 2：服务商调整客户配额
```
[Provider A] 将 Customer A1 存储配额从 10GB 提升至 50GB
    └─ [Customer A1] 立即生效，可继续上传（如已超原配额仅影响新增）
```

### 示例 3：客户超配额
```
[Customer A1] 配额已用完
    ├─ 上传接口返回 400 "存储配额不足"
    ├─ 浏览/下载仍可正常使用
    └─ 站内信自动推送"配额告警"通知
```

## 五、常见问题

### Q1. 服务商能否绕过开发商使用未开放的支付渠道？
**不能**。`ToggleChannel` 关闭后，`tenant_payment_auth` 记录同步失效，API 层强制校验。

### Q2. 终端客户能否看到其他客户的文件？
**不能**。PG RLS + GORM 所有查询都会带 `tenant_id = current_tenant_id()` 过滤。即使应用层漏判，DB 层也会兜底拒绝。

### Q3. 权限变更需要用户重新登录吗？
**不需要**。权限缓存 5 分钟自动失效，且可通过管理接口手动清除。

### Q4. 如何审计敏感操作？
所有管理后台接口都记录结构化访问日志，含 `trace_id / user_id / tenant_id / method / path / status`，可按时间区间过滤。
